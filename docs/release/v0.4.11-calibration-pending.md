# v0.4.11 — Opus 4.8 재캘리브레이션 (PENDING)

**Status**: PENDING measurement — ADR-009 §7
**Date**: 2026-05-29

## 왜 이 문서가 있는가

forgen 의 효과 크기 주장(δ > 0, "forgen 이 vanilla baseline 대비 통계적으로 유의미한
개선")은 **v0.4.5 에서 sonnet/codex 드라이버 기준으로** 측정됐다 (README 의
"mean δ = +0.151 W, 95% CI [+0.118, +0.184]" 등).

Claude Opus 4.8 은:
- effort 기본값 high + xhigh/ultracode 도입,
- "코드 결함 4× 덜 놓침 + 자기 오류 캐치 + 불건전 계획 반박" 을 **네이티브로** 제공.

즉 baseline(forgen 없는 4.8)이 이전보다 강해졌다. 따라서 **sonnet/codex 의 δ 가
opus-4.8 에서도 유지된다고 가정할 수 없다**. forgen 의 메타 가드 거짓양성 비용도
상승했을 수 있다(이 세션에서 L1·dangerous-pattern 자기참조 오탐 2건 관찰).

## 규칙 (프로젝트 룰)

> opus-4.8 baseline 에 대한 재측정 전까지, 릴리스 노트/README/마케팅에서
> **"opus-4.8 에서 δ > 0"** 또는 그에 준하는 효과 주장을 하지 않는다.
> 기존 δ 주장은 "sonnet/codex 기준" 이라는 단서를 유지한다 (README 는 이미 명시).

숫자를 추정/날조해 채우지 않는다 — 실제 N-run 측정만 유효.
