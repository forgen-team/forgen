# v0.4.4 Release Checklist — subscription-mode evidence

**Status**: in-progress (2026-04-30 measurement). Internal evidence release, NOT v0.5.0 release-proof.

This is a **scaled-down** measurement compared to `v0.5.0-checklist.md`. The original v0.5.0 plan required local 64GB+ RAM judges (Qwen 72B + Llama 70B), which the development environment cannot host. v0.4.4 substitutes **subscription-mode CLI judges** (`claude` CLI haiku + `codex` CLI gpt-5-codex) — different model families = κ-independent.

## Why v0.4.4, not v0.5.0

- v0.5.0 PUBLIC track required local-only reproducibility for external claims. Without 70B local judges, that claim cannot be made.
- Subscription-mode evidence is **internal-only** signal — sufficient for "does forgen mechanism work" but not for "third parties can reproduce".
- Full v0.5.0 release-proof remains DEFERRED until 64GB local environment is available (e.g., dedicated workstation, A100 cloud rental, or alternative reproducibility strategy).

## Run Procedure (v0.4.4)

```bash
# Subscription-mode ψ statistical run (N=10, ~1.5-2h with llama3.1:8b driver)
cd packages/forgen-eval
PSI_STAT_N=10 \
  JUDGE_TRACK=API_DEV \
  FORGEN_EVAL_DATA_DIR=/tmp/forgen-eval-data \
  OLLAMA_DRIVER_MODEL=llama3.1:8b \
  OLLAMA_DRIVER_MAX_TOKENS=256 \
  npx tsx src/runners/demo-psi-stat-judged.ts

# Output: reports/psi-stat/psi-stat-judged-API_DEV-<ts>.json
# Inspect: mean ψ, 95% bootstrap CI, κ between judges
```

### Driver speed notes
- qwen2.5:14b: ~2.2 tok/s on Apple Silicon CPU/Metal → 512 tokens ~232s/call
- llama3.1:8b: ~4.4 tok/s same hardware → 256 tokens ~58s/call (4x faster end-to-end)
- Each case: 4 arms × 2 chat calls (turn 1 + trigger) ≈ 8 driver calls + 8 parallel judge pairs
- **Recommended driver = llama3.1:8b + max_tokens=256** (env vars above)
