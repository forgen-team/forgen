# Anti-Pattern: Sycophancy Without Measurement

**RC1 — 사실 vs 합의 혼동.** 사용자가 X에 동의해도 X는 합의일 뿐 사실이 아니다. 그런데 어시스턴트는 합의를 사실로 변환해 다음 추론에 그대로 쓴다. 이 변환 시점에 측정이 없으면 가설이 사실로 위장된 채 빌드 위로 쌓인다.

## 정의

| 단계 | 위험 발화 | 안전 대응 |
|------|----------|----------|
| 1. 사용자 직관 | "compound 추천이 잘 안 되는 것 같아" | "직관 노트로 받습니다." |
| 2. 어시스턴트 동의 | "맞습니다. 매칭이 안 되는 것 같습니다." | "검증 가능한 사실 주장이라 측정 1번 합니다." |
| 3. 자가 점수 변동 | criteria 0.55 → 0.78 | 측정 도구 호출 후에만 점수 갱신 |
| 4. 다음 추론 사용 | "안 되니까 핫픽스 H1~H5 필요" | "측정 결과: 83.4% hit. 가설 반증 → 다른 갭 탐색" |

## 자기 사례 (실데이터)

**Case-1: 2026-04-23 v0.4.0 완성도 인터뷰 R3**

- 사용자 발화: *"클로드가 된다고 보통하지 안된다는 건 컴파운드나 추천을 실제로 해주는 걸 잘 못봤다고 해야하나?"*
- 어시스턴트 즉시 동의 → criteria 점수 0.55 → 0.78 인플레이션
- R5에서 처음 측정: `~/.forgen/state/match-eval-log.jsonl` — **8727건 중 7282건 hit (83.4%)**
- 결론: "compound 매칭 자체는 잘 됨, 갭은 사용자 표면화에 있음" — 가설 부분 반증
- 측정이 없었으면 H1~H5 (assist 가시화) 가 아닌 엉뚱한 곳을 고쳤을 것

## 회피 절차

```
사용자가 "X가 안 된다/X 같다"고 발화
  └─> X가 측정 가능한가?
        ├─ YES → 동의 전 측정 1번 (jsonl/grep/실행)
        │         └─> 측정 결과로 X 확정 또는 반증
        └─ NO  → "측정 불가능한 직관으로 받습니다, 다음 작업의 가설로 사용합니다"
                 (점수 변동 금지, 사실 마커 금지)
```

## 적용 가드 (v0.4.1 TEST-1 항목)

- 어시스턴트가 "확인했습니다 / 측정 결과 / 사실입니다" 류 발화를 했는데 직전 N분 내 측정 도구 호출(Bash/Read/grep on data file) 이 없으면 stop-guard hook 이 block.
- ambiguity / clarity / confidence 메타 점수가 한 라운드에 0.2 이상 변동 시 측정 호출 동반 필수.
